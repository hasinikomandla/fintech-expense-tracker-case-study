# Fintech-expense-tracker-case-study
UI/UX case study for a personal finance and expense tracking mobile application, covering user research, UX strategy, and high-fidelity mobile screens.

A mobile application designed to help users track expenses, monitor budgets, and gain insights into their spending behavior through a clean and intuitive user experience.

---

##  Project Overview

Managing personal finances can be challenging due to the increasing number of daily digital transactions. Many users struggle to understand where their money is being spent, how much budget remains for the month, and which spending categories contribute most to their expenses.

FinTrack was designed to simplify expense tracking by providing clear financial summaries, quick expense entry, organized transaction history, and visual spending insights. The goal was to create a clean and intuitive experience that helps users make informed financial decisions with minimal effort.

---

##  Problem Statement

Many users struggle to:

- Understand where their money is being spent
- Track expenses consistently
- Monitor their remaining monthly budget
- Identify major spending categories
- Gain meaningful insights from financial data

The objective was to design a solution that simplifies expense management while helping users make better financial decisions.

---

##  Target Users

- Students managing personal budgets
- Young professionals tracking monthly expenses
- Individuals seeking better financial awareness
- Users who want a simple and organized expense tracking experience

---

##  User Flow

Splash Screen → Sign Up / Sign In → Dashboard → Add Expense → Transaction History → Analytics → Profile & Settings

The flow was designed around the primary user goal: recording expenses quickly while maintaining visibility into spending patterns and account settings.

## Design Process

### 1. Understanding the Problem
Identified common challenges users face when managing personal finances and tracking expenses consistently.

### 2. Information Architecture
Organized the application's structure to prioritize quick expense recording and financial visibility.

### 3. User Flow Design
Created a streamlined flow that reduces friction while supporting the user's primary goals.

### 4. Visual Design
Applied visual hierarchy, consistency, and usability principles to improve the overall experience.

### 5. High-Fidelity Interface Design
Designed polished mobile screens focused on clarity, simplicity, and efficient task completion.

---

##  Key Design Decisions

### Splash Screen
- Established strong brand identity
- Communicated trust and reliability
- Created a clear entry point into the application

### Authentication
- Reduced onboarding friction
- Limited forms to essential information
- Used clear call-to-action buttons

### Dashboard
- Prioritized budget visibility
- Highlighted spending progress
- Provided quick access to recent transactions

### Add Expense
- Focused on fast task completion
- Reduced manual input requirements
- Simplified category selection

### Transaction History
- Improved scanability through grouping
- Enabled quick transaction review
- Maintained visual consistency

### Analytics
- Transformed raw data into actionable insights
- Supported spending awareness
- Improved data readability through visualization

### Profile & Settings
- Centralized account management
- Supported personalization
- Provided budget and preference controls
